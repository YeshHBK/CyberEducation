What is a Cloud Attack

Cloud computing attacks target platforms that offer storage or hosting services, aiming to get access to user data or prevent them from using their cloud services. This can be done through exploiting vulnerabilities in cloud computing or taking the users credentials

According to the 2016 State of the Cloud Survey, 95% of respondents are using the cloud. Despite its rapid growth, the nature of cloud computing introduces the possibility of serious cloud security breaches that can drastically affect an organization. Data security is one of the leading concerns for IT professionals. How can IT managers protect themselves (and their organizations) while still enabling innovation, data access and flexibility? And, if you are interested in other types of security threats to organizations. Although there has been a recent buzz around vulnerabilities and malware, risks in cloud computing are due to issues with configuration and authentication as of lately. As the world is finding out, utilizing the cloud to host your enterprise’s data and applications can provide a multitude of benefits in the realms of management, scalability, and accessibility. Unfortunately, it is becoming more apparent that the cloud also carries some risk behind it. Typically, it is more commonly heard that risks have revolved around areas of data loss, malware, denial of service, and overall system vulnerabilities. A recent report by Cloud Security Alliance states that the most recent threats stem from cloud strategy and implementation decisions. There was a survey of over 200 industry experts that helped narrow down nine notable cloud-related threats and risks. For each of these threats, there are also recommendations on how to try and become resilient against them

  • 1) Strong Authentication – since the most common way to access user’s data is through stolen credentials, it is important to implement strong authentication methods and identity management tools. This can be done through multi-factor authentication and using tools that confirm identity through one time passwords or biometric methods
  • 2) Protect Data – at its core, the data must be kept safe so it is wise encrypt the data in the cloud environment at all stages of transfer and storage; from the source on the user’s side, during the transfer from the suer to the cloud and when stored in the cloud database
  • 3) Implement Access Management – allowing cloud users to assign role-based permissions to different admins so that the users only have the ability to access data that is assigned to them
